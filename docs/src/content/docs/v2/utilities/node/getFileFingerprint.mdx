The **`getFileFingerprint`** function generates a fingerprint for a file by hashing:

1. A sample from the beginning of the file.
2. Identity data (default file metadata or custom identity data).

It is useful for fast file change detection and deduplication workflows where reading the full file every time is unnecessary.

## Usage
```ts
import { getFileFingerprint } from '@mustib/utils/node';

const result = await getFileFingerprint({
  filePath: '/absolute/path/to/file.txt',
});

if (result.status === 'success') {
  console.log(result.data.fingerprint);
} else {
  console.error(result.data.message, result.data.context);
}
```

## Usage with custom identity data
```ts
import { getFileFingerprint } from '@mustib/utils/node';

const result = await getFileFingerprint({
  filePath: '/absolute/path/to/file.txt',
  hashAlgorithm: 'sha256',
  sampleSize: 1024,
  identityData: ({ stats }) => ({
    size: stats.size,
    createdAt: stats.birthtimeMs,
  }),
});
```

## Error behavior
- Returns `{ status: 'error' }` with message `"not a file"` when the path exists but is not a file (for example, a directory).
- Returns `{ status: 'error' }` with message `"failed to read file sample"` when stream reading fails.
- Rejects if `stat(filePath)` fails.
- Rejects if `hashAlgorithm` is invalid.
